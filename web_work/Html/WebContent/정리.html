HTML 정리

시맨틱 웹

  • 컴퓨터가 사람을 대신하여 정보를 읽고, 이해하고 가공하여 새로운 정보를 만들어 낼 수 있도록 만든 차세대 지능형 웹
  • 월드 와이드 웹의 창시자인 팀 버너스-리가 제안
  • 웹 페이지에 포함된 정보의 의미를 명확하게 하여 컴퓨터가 사람처럼 정보를 이해하고 처리할 수 있도록 하는 것을 목표

목표

  • 데이터의 재사용과 공유 촉진: 데이터를 보다 구조화하고 연결 가능하게 만들어 다양한 애플리케이션에서 쉽게 재사용하고 공유할 수 있습니다.
  • 의미 기반 검색: 데이터에 의미를 부여함으로써 검색 엔진이 더 정확하고 관련성 높은 결과를 제공할 수 있습니다.
  • 자동화된 추론: 데이터 간의 관계를 이해함으로써, 컴퓨터가 자동으로 정보를 연결하고 추론할 수 있게 됩니다.

웹 애플리케이션

  • 인터넷이나 인트라넷을 통해 웹 브라우저에서 이용할 수 있는 응용 프로그램
  • 클라이언트-서버 구조를 기반으로 하며, 사용자는 웹 브라우저(클라이언트)를 통해 서비스를 요청하고, 서버는 이러한 요청을 처리하여 결과를 사용자에게 전달
  • 대체로 HTTP/HTTPS 프로토콜을 통해 이루어짐

장점

  • 접근성: 인터넷 연결만 있으면 언제 어디서나 접근 가능
  • 유지 관리: 서버 측에서 업데이트를 하면 모든 사용자에게 즉시 반영되므로 효율적
  • 플랫폼 독립성: 다양한 운영 체제나 디바이스에서 사용 가능

단점

  • 인터넷 의존성: 인터넷 연결 없이는 사용이 불가능
  • 보안 문제: 데이터가 네트워크를 통해 전송되므로 보안에 신경 써야 함
  • 성능 제한: 로컬 애플리케이션보다는 느릴 수 있음, 특히 데이터 처리가 많은 경우

웹 표준

  • 웹 표준은 웹 페이지의 제작과 관련된 기술 사양을 정의하고, 이를 통해 웹 개발자들이 보다 접근성 높고 호환성 있는 웹 페이지를 제작할 수 있도록 도움

크로스 브라우징

  • 크로스 브라우징은 웹 사이트나 애플리케이션이 서로 다른 브라우저(예: Chrome, Firefox, Safari, Internet Explorer 등)에서 동일하게 작동하도록 보장하는 개발 방식

웹접근성

  • 웹 사이트, 도구, 기술이 장애를 가진 사람들을 포함한 모든 사람들이 이용할 수 있도록 설계 및 개발되어야 한다는 원칙
  • 정보와 기능의 사용이 모든 사용자에게 동등하게 이용 가능해야 함을 의미
  • 장애 유무, 사용 환경, 연령 등에 관계없이 모든 사람이 웹을 효율적으로 사용할 수 있도록 보장하는 것을 목표

html

  • 웹 페이지를 만들기 위한 표준 마크업 언어
  • 웹 페이지의 구조를 설명하고 웹 브라우저에게 콘텐츠를 어떻게 표시할지 지시
  • 목록 태그: ul, li, ol, dl, dt
  • 링크, 이미지: a, img
  • 그룹태그: div, span
  • 폼: form, input
  • 시멘틱: section, article, nav, footer

HTML5

  • HTML5는 HTML의 최신 버전으로, 웹 애플리케이션과 멀티미디어 콘텐츠에 대한 더 나은 지원을 제공
  • HTML을 사용하여 웹 페이지를 만들 때, 개발자는 웹 표준을 준수하여 웹 접근성을 확보하고, 다양한 브라우저와 장치에서 일관된 사용자 경험을 제공하는 것이 중요

CSS

  • 웹 페이지의 디자인과 레이아웃을 정의하는 스타일 시트 언어
  • HTML이 웹 페이지의 구조를 제공한다면, CSS는 그 구조에 색상, 글꼴, 간격 등의 시각적 스타일을 적용하여 사용자 인터페이스의 외관과 느낌을 개선

주요기능

  • 레이아웃 제어: CSS는 웹 페이지의 레이아웃을 조정하는 데 사용됩니다. 이는 텍스트의 정렬, 컬럼 레이아웃, 위치 조정과 같은 다양한 방식으로 적용될 수 있습니다.
  • 시각적 효과: 색상, 그림자, 테두리, 투명도 등을 조정하여 요소의 시각적 매력을 증가시킬 수 있습니다.
  • 타이포그래피: 글꼴, 글자 크기, 줄 간격, 텍스트 정렬 등 텍스트의 스타일을 정의할 수 있습니다.

XML

  • EXtensible Markup Language
  • 데이터를 저장하고 전송하기 위해 설계된 마크업 언어
  • 이기종간 정보 전달의 목적을 띔
  • 데이터를 계층적 구조로 표현할 수 있으며, 사용자가 태그를 자유롭게 정의할 수 있어서 다양한 유형의 데이터를 표현하는 데 적합

JSON

  • 데이터를 저장하고 교환하기 위해 개발된 경량의 데이터 형식
  • 이기종간 정보전달의 목적을 띔
  • JavaScript에서 객체를 만드는 방법을 기반으로 하여, 텍스트 기반의 구조로 데이터를 표현
  • XML보다 읽기 쉽고, 파싱이 빠르며, 네트워크를 통한 데이터 전송에 효율적

DOM

  • 웹 문서의 구조를 프로그래밍 언어가 이해하고 조작할 수 있는 방식으로 표현한 것
  • HTML의 작은 부분까지 접근할 수 있는 구조로, DOM을 이용하여 HTML로 구성된 웹페이지를 동적으로 움직이게 만들 수 있다.

DOM의 주요 특징

  • 트리 구조: DOM은 문서를 트리 구조로 모델링합니다. 이 트리는 문서의 각 요소, 속성, 내용 등을 노드로 표현하며, 이러한 노드들이 계층적인 관계를 형성합니다.
  • 언어 독립성: DOM은 어떠한 특정 프로그래밍 언어에도 종속되지 않고, JavaScript, Java, Python 등 다양한 언어에서 사용될 수 있습니다.
  • 동적 인터페이스: DOM을 사용하면 스크립트를 통해 문서의 구조, 스타일, 내용 등을 동적으로 변경할 수 있습니다. 예를 들어, 사용자의 액션에 따라 요소를 추가하거나 삭제하고, 스타일을 변경하며, 이벤트를 처리할 수 있습니다.
<!DOCTYPE html>
<html>
<head>
<meta charset="UTF-8">
<title>Insert title here</title>
</head>
<body>
    <h1>홈페이지</h1>
    <form action="서버URL" method="get/post">
        <fieldset>
            <ol>
            <li>
                <label for="id">아이디</label>
                <input type="text" name="userId" id="id" autofocus="autofocus"
                placeholder="하이요">
            </li>
            <li>
                <label>패스워드</label>
                <input type="password" name="userPass">
            </li>
            <li>
                <label>이메일</label>
                <input type="email" name="userPass" placeholder="aa@aa.com" required="required">
            </li>
            <li>
                <label>배송날짜</label>
                <input type="date">
            </li>
            <li>
                <textarea rows="5" cols="60" placeholder="지원동기"></textarea>
            </li>
            </ol>
        </fieldset>
        <fieldset>
            <input type="submit" value="전송">
        </fieldset>
    </form>
</body>
</html>
